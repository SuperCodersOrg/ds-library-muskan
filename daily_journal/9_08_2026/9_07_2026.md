# Section 1 — Specific Bug / Confusion

Today's biggest challenge was designing a command parser that works correctly for a generic `RedisLite`.

Initially, I used `std::cin` to read commands.

Although individual commands worked, entering multiple commands caused input-buffer synchronization issues.

The remaining tokens from one command were interpreted as part of the next command, producing incorrect parsing.

After switching to `std::getline()`, a new problem appeared.

Every extracted token became a string, but my `RedisLite` stores generic key and value types.

The challenge became converting those strings into arbitrary template types while still supporting string values containing spaces.

# Section 2 — Failed Attempt

My first implementation relied entirely on:

```cpp
std::cin >> value;
```

After replacing it with `std::getline()`, I used:

```cpp
std::stringstream >> value;
```

to convert strings into generic types.

This worked correctly for primitive types.

However, I discovered that `std::stringstream` behaves exactly like `std::cin`.

For example:

```text
SET 10 Hello World
```

The value extracted became only:

```text
Hello
```

because extraction stops at whitespace.

To solve this, I implemented a generic converter function and created a template specialization for `std::string` that returns the complete string instead of extracting only the first word.

This allowed both primitive and string values to be handled correctly.

# Section 4 — Learning Reflection

Today strengthened my understanding of command-line application design and generic programming.

I learned that reading input token-by-token using `std::cin` is not suitable for interactive command interpreters because leftover input can affect later commands. Processing one complete line at a time with `std::getline()` provides much better control over parsing.

I also learned how generic type conversion can be implemented using templates and `std::stringstream`, and why template specialization is necessary for types such as `std::string`. Primitive types and strings require different conversion strategies because formatted extraction stops at whitespace.

By the end of the day, I completed the first functional version of the `RedisLite` interface and gained a much deeper understanding of parsing, template specialization, and generic input handling.
